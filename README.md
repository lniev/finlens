# 直播录制插件

一个浏览器插件，用于自动录制直播页面的音频和视频内容（分开录制），并保持页面静音状态。

## 功能特性

- 🎵 自动录制音频内容（单独保存为audio_*.webm文件）
- 🎥 自动录制视频内容（单独保存为video_*.webm文件）
- 🔊 页面保持静音状态，不影响录制音频
- 📁 音频和视频分开保存，便于后期处理
- 🎛️ 可配置录制选项
- 📱 简单易用的用户界面

## 支持的浏览器

- Google Chrome (推荐)
- Microsoft Edge (推荐)
- Mozilla Firefox (有限支持)

## 快速开始

### 1. 准备图标文件

由于Chrome插件需要图标文件，您需要：

1. 将`icon.svg`转换为PNG格式的三个尺寸文件（16x16, 48x48, 128x128）
2. 或者使用`icon-generator.html`（见下文）快速生成
3. 文件命名为：`icon16.png`, `icon48.png`, `icon128.png`

### 2. 安装插件

#### Chrome/Edge浏览器

1. 打开Chrome/Edge浏览器
2. 输入`chrome://extensions/`（Chrome）或`edge://extensions/`（Edge）到地址栏
3. 开启"开发者模式"（右上角开关）
4. 点击"加载已解压的扩展程序"
5. 选择插件文件夹（包含manifest.json的目录）

#### Firefox浏览器

1. 打开Firefox浏览器
2. 输入`about:debugging`到地址栏
3. 点击"此Firefox"
4. 点击"临时加载附加组件"
5. 选择插件文件夹中的manifest.json文件

## 使用方法

1. 打开任意直播页面
2. 点击工具栏中的插件图标
3. 确认录制选项：
   - ✅ 录制音频（默认开启）
   - ✅ 录制视频（默认开启）
   - ✅ 页面静音（默认开启）
4. 点击"开始录制"按钮
5. 在弹出的屏幕共享对话框中，选择"标签页"，然后选择当前直播页面
6. **重要**：确保勾选"分享音频"选项（Chrome需要）
7. 点击"分享"开始录制
8. 录制过程中页面会自动静音，但音频仍然会被录制
9. 点击"停止录制"按钮结束录制
10. 录制的音频和视频文件会自动下载到浏览器默认下载目录

## 生成图标

使用`icon-generator.html`生成PNG图标：

1. 在浏览器中打开`icon-generator.html`
2. 依次点击三个"下载"按钮
3. 将下载的文件保存为：
   - `icon16.png` (16x16)
   - `icon48.png` (48x48)
   - `icon128.png` (128x128)

## 技术实现

### 录制原理

- 使用`MediaRecorder` API捕获标签页内容
- 使用`getDisplayMedia`获取音频和视频轨道
- 创建独立的MediaRecorder实例分别录制音频和视频
- 使用`Blob`对象存储录制数据
- 自动下载录制完成的文件

### 静音机制

- 监听页面上的media元素
- 自动静音所有video和audio元素
- 使用MutationObserver监听新添加的媒体元素
- 保存原始音量状态，录制结束后可恢复

## 配置选项

- **录制音频**: 是否录制音频内容（默认开启）
- **录制视频**: 是否录制视频内容（默认开启）
- **页面静音**: 录制时是否静音页面（默认开启）

## 注意事项

### 重要提示

1. **必须选择"标签页"共享**：屏幕共享时请选择"标签页"而不是"整个屏幕"，以确保能捕获音频
2. **必须勾选"分享音频"**：Chrome需要勾选"分享音频"才能录制系统音频
3. **页面静音不影响录制**：插件会静音页面，但录制的音频不受影响
4. **文件格式**：录制文件为WebM格式，可以使用VLC播放器或ffmpeg转换

### 常见问题

#### 无法录制音频

- 确保在屏幕共享时勾选了"分享音频"
- 检查是否选择了"标签页"而不是"整个屏幕"
- 某些浏览器可能不支持音频捕获

#### 录制视频没有声音

- 确保已勾选"录制音频"选项
- 确认屏幕共享时已勾选"分享音频"
- 检查浏览器是否支持标签页音频捕获

#### 页面没有静音

- 确保已勾选"页面静音"选项
- 刷新页面后重试
- 检查浏览器控制台是否有错误

#### 无法下载文件

- 检查浏览器下载权限
- 检查磁盘空间是否充足
- 尝试手动下载（某些情况下可能需要）
